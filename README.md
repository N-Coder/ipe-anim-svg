# Ipe SVG Processing Tools

This repository contains tools for converting Ipe documents to SVG and post-processing the resulting SVG files to apply class tags and restructure the page layout.
The example directory contains a converted example presentation and how it can be included in an interactive HTML page that step-by-step displays the presentation with animated transitions. 

## Tools

### ipe_svg

`ipe_svg` is a C++ application that renders an Ipe (`.ipe`) document into an SVG file using the Cairo graphics library. While rendering, it outputs a list of class tags for each object it draws to `stdout`.

- **Input**: An Ipe document (`.ipe`).
- **Output**: 
    - An SVG file (e.g., `test.svg`).
    - A sequence of space-separated tags on `stdout`, one line per rendered object.
- **Tags**: Each line of output contains metadata about the object, such as:
    - `objnr-X`: The object's index on the Ipe page.
    - `layernr-X`: The layer index.
    - `layer-NAME`: The name of the layer.
    - `page-X`: The original Ipe page number.
    - `type-TYPE`: The object type (e.g., `type-path`, `type-text`, `type-group`).
    - `custom-DATA`: Any custom data associated with the object in Ipe.

### apply_classes.py

`apply_classes.py` is a Python script that takes the SVG generated by `ipe_svg` and the tags from its `stdout` to enhance and restructure the SVG.

- **Class Application**: It matches each line of tags to the corresponding object in the SVG (which are initially wrapped in `<page>` elements by `ipe_svg`).
- **Structure Optimization**:
    - If an object wrapper contains multiple children, it is converted to a group (`<g>`) with the tags applied as a `class` attribute.
    - If it contains only a single child, the wrapper is removed, and the `class` attribute is applied directly to that child.
- **Regrouping**: After processing individual objects, the script regroups them into new `<page>` elements. A new page is started whenever the `page-X` tag changes, effectively recreating the page structure from the original Ipe document while keeping the objects tagged with their specific metadata.

### ipe_info.py

`ipe_info.py` is a Python script that parses an Ipe XML file to extract structural information about pages, layers, and views.

- **Input**: An Ipe document (`.ipe`).
- **Output**: A JSON array of page objects. Each object contains:
    - `layers`: A list of all layer names in the page.
    - `views`: A mapping of view indices (1-based, as strings) to a list of layers visible in that view.
- **Purpose**: This metadata is essential for implementing view-based navigation in an SVG viewer, as it defines which layers should be visible for each step (view) of a page.

## Usage Workflow

1. **Build `ipe_svg`**:
   ```bash
   cmake -B build -G Ninja
   cmake --build build
   ```

2. **Generate SVG and Tags**:
   ```bash
   ./build/ipe_svg input.ipe test.svg > tags.txt
   ```

3. **Apply Classes and Regroup**:
   ```bash
   python3 apply_classes.py test.svg tags.txt output.svg
   ```

4. **Extract Metadata**:
   ```bash
   python3 ipe_info.py input.ipe > info.json
   ```

The final `output.svg` and `info.json` together provide all the information needed to build a presentation-style viewer for the original Ipe document.
